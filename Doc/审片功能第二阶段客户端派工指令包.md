# 审片功能第二阶段客户端派工指令包

## 1. 文档用途

本文件用于把“导演工作台完成动作与完成校验”这一阶段的客户端工作直接派发给其他 AI 或开发执行。

目标目录：`D:\project\MovtoolsEx\movtools-client\`

参考文档：

- `D:\project\MovtoolsEx\Doc\项目固化文档.md`
- `D:\project\MovtoolsEx\Doc\通信协议文档.md`
- `D:\project\MovtoolsEx\Doc\导演账号功能设计文档.md`
- `D:\project\MovtoolsEx\Doc\审片功能第二阶段服务器派工指令包.md`

---

## 2. 本阶段目标

在导演工作台中加入明确的 `审片完成` 动作，并按正式审片镜头完成情况做禁用校验。

本阶段完成后必须满足：

1. 导演逐镜头仍可做“镜头通过”或“提交反馈”
2. 新增任务级 `审片完成` 按钮
3. 只有全部 `正式审片镜头` 都已处理时，才允许点击完成
4. `上下文陪审镜头` 不纳入完成阻塞条件

---

## 3. 工作边界

1. 本阶段聚焦导演工作台，不改制片归档视图
2. 完成按钮依赖服务端第二阶段状态机，但前端仍需做明确禁用提示
3. 本阶段不处理关闭任务后的手动删除反馈入口

---

## 4. 推荐重点文件

- `src/pages/ReviewPage.tsx`
- `src/repositories/remote/reviewRepository.ts`
- `src/types/review.ts`
- 与任务统计、镜头队列状态、按钮禁用原因相关 helper / store

---

## 5. 必做事项

1. 在导演工作台中增加独立的 `审片完成` 按钮，不与单镜头 `镜头通过` 复用
2. 定义前端完成判定：只统计 `reviewParticipationMode === review` 的镜头
3. 对每个正式审片镜头，已处理的判定为：已 `DIRECTOR_APPROVED`，或存在正式反馈事实
4. `context` 镜头只参与连播，不参与完成判定、待处理计数和禁用提示
5. 在 UI 中明确提示“还有哪些正式审片镜头未处理”或给出总数提示
6. 完成后任务应从导演主待办区移出，进入已完成列表的前置显示模型可先占位

---

## 6. 禁止事项

1. 不要把“当前镜头可通过”误当成“整任务可完成”
2. 不要把 `context` 镜头算进未完成数
3. 不要跳过服务端完成接口直接本地改状态

---

## 7. 验收标准

1. 只要还有一个正式审片镜头既没通过也没反馈，就不能点 `审片完成`
2. 所有正式审片镜头都已处理后，`审片完成` 可点击
3. `context` 镜头不会阻塞完成
4. 导演任务主工作区的状态提示清晰
5. `npm run typecheck` 与 `npm run build` 通过

---
