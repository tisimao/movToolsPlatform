# MovtoolsEx 文档索引

本文档是 `Doc` 目录的入口，也是项目交接的第一阅读页。以仓库实际代码和当前部署方式为准；历史阶段性派工包已移出正式基线，后续只把最终结论固化到正式文档。

## 当前项目现状

- 客户端：Electron 37 + React 19 + TypeScript + electron-vite + Zustand。
- 服务端：ASP.NET Core / .NET 9 + EF Core + PostgreSQL + SignalR + JWT。
- 部署：服务端支持 Docker Compose，本地开发默认 API 端口为 `5001`。
- 业务形态：已进入“制片 - 导演 - 制作人员”三角色内部审片闭环。
- 服务端：只保存协同主数据、审片事实和权限关系，不做本机文件扫描、预览、转码、Layout 引用检查。
- 客户端：负责本机文件扫描、路径映射、预览、FFmpeg、提取、串片、本地 SQLite 能力。

## 按角色看功能

- 管理员：账号、角色、协同路径/存储根、全局配置和排障。
- 制片：项目、成员、集、镜头、任务装配、提交与关闭。
- 导演：审片任务、反馈卡片、通过/返修、完成任务。
- 制作人员：镜头查看、反馈消费、本机文件能力、按反馈修改。

## 文档维护规则

1. 以代码实际实现为准，文档描述不得覆盖或假设不存在的能力。
2. 接口、状态、字段以 `通信协议文档.md` 为主；产品边界以 `项目固化文档.md` 为主。
3. 部署相关文档需要明确使用场景：开发测试、准生产、群晖专项或 GitHub/GHCR 发布。
4. 阶段性任务完成后，只把最终规则固化到正式文档，不长期保留过程派工包。
5. 涉及账号、密钥、数据库密码的内容只能保留示例或初始化提醒，不写入真实生产凭据。
6. 路径配置相关文档统一使用“项目根目录”“镜头文件根目录”“Layout 根目录”口径；不得再把旧的默认 MA 根目录、默认视频根目录写回正式文档。
