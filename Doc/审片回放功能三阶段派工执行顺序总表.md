# 审片回放功能三阶段派工执行顺序总表

## 1. 文档用途

本文件用于统一说明“审片回放功能三阶段改造”三份派工指令包的推荐执行顺序、依赖关系与派发方式，便于直接按顺序交给 `gpt-5.4-mini` 或其他执行方。

- `D:\project\MovtoolsEx\Doc\审片回放功能第三阶段客户端派工指令包.md`

---

## 2. 总体执行原则

1. 三个阶段都只改客户端，不改服务端协议
2. 三个阶段都严禁把镜头详情页问题转嫁为导演审片页主流程改造
3. 前一阶段未稳定前，不要跳到后一阶段
4. 每一阶段结束后都必须输出修改文件、规则变化、自测结果、剩余风险

---

## 3. 推荐执行顺序

### 阶段一：数据源收口

1. `Doc/审片回放功能第一阶段客户端派工指令包.md`

目标：镜头详情页导演反馈分页只使用统一反馈接口结果，并按“当前版本 + 当前轮”生成同源卡片与 timeline。

### 阶段二：逐帧回放化

2. `Doc/审片回放功能第二阶段客户端派工指令包.md`

目标：镜头详情页导演反馈回放按当前视频帧动态显示正式绘制内容。

### 阶段三：一致性收口

3. `Doc/审片回放功能第三阶段客户端派工指令包.md`

目标：镜头详情页导演反馈回放与导演审片页正式回放结果一致，并完成多场景回归。

---

## 4. 每阶段派发建议

1. 一次只派发一份指令包给执行方，避免上下文污染
2. 明确要求执行方不要修改 `src/pages/ReviewPage.tsx` 的导演审片编辑主流程
3. 若阶段二或阶段三必须读取 `ReviewPage.tsx`，只允许做“对照分析”，不允许顺手改动主链路
4. 每阶段交付时都必须附带“本阶段没有触碰导演审片稳定链路”的说明
