## 2. 客户端总目标

全部阶段完成后，客户端必须具备：

1. 导演账号导航、默认落点、仪表盘入口与审片工作台主链路
2. 审片工作台纵向结构、任务镜头队列、三色状态、播放器进度条与底部反馈区
3. 暂停态绘制、本地绘制数据保存、清屏打断继承、历史帧修改联动更新
4. 反馈说明草稿列表、编辑/删除/回跳、镜头通过互斥规则
5. `提交反馈` 时按镜头提交当前轮全部文字反馈、全部本地绘制数据与镜头状态
6. 镜头详情页 `导演反馈` 分页中的遮罩开关与正式回放能力

---

## 3. 必须遵守的客户端边界

1. 绘制与文字反馈弱绑定
2. 允许只提交文字反馈，不允许只提交绘制
3. 绘制底层按单帧记录，本地显示层按继承链解释
4. `清屏` 是当前帧动作，会打断此前绘制继承
5. 本地绘制数据作用域属于当前镜头
6. 切镜头、刷新页面、关闭页面时，本地绘制数据必须保留
7. `提交反馈` 的绘制提交边界是“当前镜头当前轮次全部本地绘制数据”
8. 镜头详情遮罩只显示当前轮正式已提交内容
9. 当前轮提交成功后，本地反馈列表与本地绘制数据清空

---

## 4. 阶段拆分

### 4.1 第一阶段：导演入口与主链路收口

目标：先把导演账号入口、导航、仪表盘与审片任务进入路径收口到正确形态。

必须完成：

1. 导演左侧主导航收口为 `仪表盘 / 镜头 / 审片`
2. 导演登录默认落到导演仪表盘
3. 导演仪表盘展示：我参与的项目、当前激活项目的审片任务、最近反馈记录
4. 仪表盘点击 `开始审片 / 继续审片` 后，可直接进入 `ReviewPage` 并自动选中对应任务
5. 导演镜头页保留为辅助查询入口，不再承担主编辑入口职责

建议重点排查：

- `src/App.tsx`
- `src/auth/permissions.ts`
- `src/pages/DashboardPage.tsx`
- `src/pages/DirectorDashboardPage.tsx`
- `src/pages/ReviewPage.tsx`
- `src/stores/*`

完成定义：

1. 导演主链路已固定为 `仪表盘 -> 审片任务 -> 审片工作台`
2. 导演不会再看到旧的通用项目管理菜单

### 4.2 第二阶段：审片工作台结构与镜头状态交互收口

目标：先把工作台结构、镜头队列、按钮规则和页面布局调到设计文档要求。

必须完成：

1. 工作台采用“上任务镜头列表、中播放器、下反馈区”的纵向结构
2. 镜头队列展示反馈数量、状态与三色逻辑：未匹配红、已匹配待处理黄、已处理绿
3. 播放器底部始终显示进度条
4. `制作人员 / 版本 / 当前帧` 信息卡下移到页面底部或 `测试审片` 上方
5. `标注说明` 文案改为 `反馈说明`
6. 底部右侧列表改为 `草稿与已提交反馈`
7. `保存为草稿`、`提交反馈`、`镜头通过` 规则落地
8. 存在反馈记录时 `镜头通过` 不可点击；反馈列表为 0 时才可点击
9. 移除 `取消镜头返修` 按钮

建议重点排查：

- `src/pages/ReviewPage.tsx`
- `src/components/review/*`
- `src/styles.css`

完成定义：

1. 页面结构与交互入口已与设计文档一致
2. 未接入绘制前，页面也能正确承载反馈列表与镜头状态交互

### 4.3 第三阶段：本地绘制数据模型与时间轴继承显示

目标：把导演绘制从“当前画一下”升级为“当前镜头下可恢复、可继承、可清屏打断”的本地数据模型。

必须完成：

1. 暂停态显示绘制工具；播放态禁止绘制
2. 同一帧支持多个绘制对象，保存为该帧绘制集合
3. 本地绘制数据作用域归属当前镜头
4. 切镜头、刷新、关闭页面后，本地绘制数据可恢复
5. 显示层按继承链解释：后一帧无新绘制时沿用最近一次有效绘制
6. `清屏` 作为当前帧动作写入本地数据，并打断此前继承
7. 回到历史帧修改绘制时，后续到下一绘制帧或清屏帧之前的显示结果自动更新
8. 本地数据结构中明确区分：反馈草稿列表、单帧绘制记录、清屏帧记录

建议重点排查：

- `src/pages/ReviewPage.tsx`
- `src/components/**/*canvas*`
- `src/lib/**/*`
- `src/types/review.ts`
- `src/stores/*`

完成定义：

1. 绘制已具备稳定本地恢复能力
2. 不生成文字反馈时也不会丢失绘制内容

### 4.4 第四阶段：反馈草稿、正式提交与本地清空

目标：打通“文字反馈为主体，绘制为增强层”的正式提交链路。

必须完成：

1. `保存为草稿` 生成当前镜头反馈列表项，不上传服务器
2. 列表支持编辑、删除、回跳到对应帧
3. 编辑反馈时，播放器跳回对应帧，并回填当前反馈说明
4. 允许只有文字反馈，不允许只有绘制提交
5. 若当前镜头只有绘制没有文字反馈，点击 `提交反馈` 时提示“尚未生成文字反馈”
6. 点击 `提交反馈` 时，按镜头上传：
   - 当前轮全部文字反馈
   - 当前轮全部本地绘制数据
   - 镜头状态
7. 点击 `镜头通过` 时，只上传镜头通过状态
8. 当前轮提交成功后，清空该镜头本地反馈列表与本地绘制数据
9. 提交失败时，本地反馈与本地绘制都必须保留，允许重试

建议重点排查：

- `src/repositories/remote/reviewRepository.ts`
- `src/repositories/types.ts`
- `src/services/repositoryService.ts`
- `src/pages/ReviewPage.tsx`
- `src/stores/*`

完成定义：

1. 提交边界与设计文档一致
2. 成功后进入下一轮空白编辑态，失败后不丢内容

### 4.5 第五阶段：镜头详情页导演反馈遮罩回放与联调验收

目标：让制作人员能在镜头详情页清晰回看当前轮反馈与遮罩视频。

必须完成：

1. 镜头详情页 `导演反馈` 分页支持视频遮罩开关
2. 开关关闭时播放原视频
3. 开关开启时，仅叠加当前轮正式已提交绘制内容
4. 遮罩不显示本地未提交绘制
5. 文字反馈列表与视频遮罩并列存在，但不强制一一对应
6. 当前轮反馈提交后，镜头详情可稳定回读本轮反馈与遮罩
7. 重新登录、切换账号、切换设备后，正式反馈与遮罩仍可回看

建议重点排查：

- `src/pages/LensPage.tsx`
- `src/components/lens/*`
- `src/services/repositoryService.ts`
- `src/styles.css`

完成定义：

1. 镜头详情已具备“看文字 + 看遮罩视频”的完整回看闭环
2. 客户端最终表现与 `导演账号功能设计文档.md` 对齐
