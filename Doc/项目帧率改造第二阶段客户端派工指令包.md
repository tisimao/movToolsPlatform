# 项目帧率改造第二阶段客户端派工指令包

## 1. 文档用途

本文件用于把“制片创建项目时可编辑项目默认 fps”这一阶段的客户端工作直接派发给其他 AI 或开发执行。

目标目录：`D:\project\MovtoolsEx\movtools-client\`

参考文档：

- `D:\project\MovtoolsEx\Doc\项目固化文档.md`
- `D:\project\MovtoolsEx\Doc\通信协议文档.md`
- `D:\project\MovtoolsEx\Doc\项目帧率改造第一阶段客户端派工指令包.md`

---

## 2. 本阶段目标

本阶段只解决“项目创建页增加 fps 可编辑输入并向创建链路透传”的问题。

完成后必须满足：

1. 制片创建项目时可看到 fps 输入项
2. 该输入项默认值为 `30`
3. 客户端创建项目请求类型、表单状态、仓储调用链均可透传该字段
4. 不改导演审片编辑主流程

---

## 3. 工作边界

1. 本阶段只改客户端表单与客户端请求透传
2. 若服务端尚未接收该字段，可先保证客户端结构完整，不擅自伪造服务端成功持久化
3. 本阶段不负责数据库落库
4. 本阶段不负责历史项目回填

---

## 4. 推荐重点文件

- `src/pages/ProjectPage.tsx`
- `src/repositories/types.ts`
- `src/repositories/remote/projectRepository.ts`
- `src/repositories/remote/types.ts`
- `src/services/repositoryService.ts`
- `src/types/project.ts`
- `src/types/ipc.ts`

---

## 5. 必做事项

1. 在创建项目表单中新增“项目默认帧率 / playback fps”输入项
2. 默认值设为 `30`
3. 对输入做基础校验，避免空值、非数字、非法值直接进入请求
4. 将该字段从页面状态透传到项目仓储 createProject 请求
5. 明确当前阶段该字段是否仅前端透传、还是已被服务端正式回读；交付说明必须写清

---

## 6. 禁止事项

1. 禁止在本阶段擅自修改服务端协议文件并假设服务端已支持
2. 禁止修改 `ReviewPage.tsx` 导演审片编辑主流程
3. 禁止把 fps 输入做成仅界面显示、但不进入请求链路的假字段
4. 禁止把默认值再退回 24

---

## 7. 验收标准

1. 创建项目表单可编辑 fps，默认值为 `30`
2. 客户端 createProject 链路请求结构中已包含 fps 字段
3. 类型定义与页面状态一致，不存在 any/临时绕过
4. `npm run typecheck` 与 `npm run build` 通过

---
