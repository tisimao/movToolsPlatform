# 审片回放功能第一阶段客户端派工指令包

## 1. 文档用途

本文件用于把“镜头详情页导演反馈回放链路收口”这一阶段的客户端工作直接派发给其他 AI 或开发执行。

目标目录：`D:\project\MovtoolsEx\movtools-client\`

参考文档：

- `D:\project\MovtoolsEx\Doc\项目固化文档.md`
- `D:\project\MovtoolsEx\Doc\通信协议文档.md`
- `D:\project\MovtoolsEx\Doc\镜头详情页设计文档.md`
- `D:\project\MovtoolsEx\Doc\导演账号功能设计文档.md`

---

## 2. 本阶段目标

本阶段只解决“镜头详情页导演反馈分页的数据真相源收口”，不重做导演审片主流程。

完成后必须满足：

1. 镜头详情页 `导演反馈` 分页只使用 `reviewService.listReviewFeedbacks(lensId)` 返回结果作为反馈与绘制真相源
2. 镜头详情页不再混用 `activeDetail.directorFeedbacks` 作为该分页的视频遮罩来源
3. 当前版本的反馈卡片列表与回放所消费的反馈集合来自同一份过滤结果
4. 不修改导演审片页的绘制、提交、草稿、画布交互流程

---

## 3. 工作边界

1. 本阶段只做镜头详情页导演反馈分页的数据收口
2. 本阶段不修改 `ReviewPage.tsx` 中导演审片工作台的绘制流程
3. 本阶段不改服务端接口语义，不新增接口
4. 本阶段不追求与导演审片页逐帧完全一致，只先保证镜头详情页使用正确的反馈数据集

---

## 4. 推荐重点文件

- `src/pages/LensPage.tsx`
- `src/lib/reviewFrameFeedback.ts`
- `src/lib/directorFeedback.ts`
- 如确有必要再读：`src/repositories/remote/reviewRepository.ts`

---

## 5. 必做事项

1. 梳理镜头详情页 `导演反馈` 分页当前真实数据链路，明确哪些状态来自 `reviewFeedbacks`，哪些来自 `activeDetail.directorFeedbacks`
2. 将该分页的卡片列表、当前轮选择、绘制 timeline 推导全部统一到 `reviewService.listReviewFeedbacks(lensId)` 的结果上
3. 基于“当前版本 + 当前轮”重新推导镜头详情页使用的反馈集合与回放 timeline
4. 保证点击反馈条目后使用的帧号与当前回放数据集来自同一批反馈
5. 清理镜头详情页中与 `directorFeedbacks` 混用导致歧义的旧状态或回退分支

---

## 6. 禁止事项

1. 禁止修改 `src/pages/ReviewPage.tsx` 的导演审片绘制流程
2. 禁止修改导演反馈提交链路、草稿链路、画布保存链路
3. 禁止把镜头详情页问题通过篡改导演审片工作台行为来规避
4. 禁止继续在镜头详情页同时维护两套“当前轮反馈真相源”
5. 禁止直接依赖 `latestRound` 作为“当前版本当前轮”的唯一真相源，除非你先证明它已经过当前版本过滤

---

## 7. 验收标准

1. 打开有导演反馈的镜头详情页时，`导演反馈` 分页中的卡片列表来自当前版本反馈
2. 回放组件消费的 timeline 与卡片列表来自同一批“当前版本当前轮”反馈
3. 当镜头同时存在多个版本反馈时，不会再错误取到其他版本的最新轮
4. `ReviewPage.tsx` 不发生业务行为回退
5. `npm run typecheck` 与 `npm run build` 通过
