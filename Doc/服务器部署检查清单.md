# Movtools Server 部署检查清单

这份清单用于帮助你在 Windows 备用机器和群辉 NAS 上逐项核对部署状态。

建议使用方法：

- 部署前先从上往下看一遍
- 实际操作时每完成一项就打勾
- 如果出问题，先回到“启动失败排查清单”逐项检查

---

## 1. 通用准备清单

在 Windows 和群辉 NAS 部署前，都先确认下面这些项目：

- [ ] 已准备好完整的 `movtools-server` 文件夹
- [ ] 已确认部署目录中包含 `src` 文件夹
- [ ] 已确认部署目录中包含 `docker-compose.yml`
- [ ] 已确认部署目录中包含 `.env` 或准备从 `.env.example` 复制
- [ ] 已准备好数据库密码 `DB_PASSWORD`
- [ ] 已准备好长度不少于 32 个字符的 `JWT_SIGNING_KEY`
- [ ] 已知道服务器所在局域网 IP 的查看方法
- [ ] 已确认客户端要连接的地址是 `http://服务器IP:5001`

---

## 2. `.env` 文件检查清单

如果你还没创建 `.env`，请先复制：

- `movtools-server/.env.example`

复制后的目标文件应为：

- `movtools-server/.env`

检查项：

- [ ] `.env` 与 `docker-compose.yml` 在同一目录
- [ ] `.env` 文件名没有写错，不是 `.env.txt`
- [ ] `DB_PASSWORD` 已改成你自己的密码
- [ ] `JWT_SIGNING_KEY` 已改成你自己的密钥
- [ ] `JWT_SIGNING_KEY` 长度不少于 32 个字符
- [ ] 每一行末尾没有多余空格
- [ ] 文件里没有中文引号
- [ ] 没有把密码外面再套引号

标准示例：

```env
DB_PASSWORD=MovtoolsDb_2026_Test!
JWT_SIGNING_KEY=MovtoolsJwtKey_ForLanTesting_2026_ReallyLong
```

---

## 3. `docker-compose.yml` 检查清单

请确认当前实际使用的是仓库中的这个文件：

- `movtools-server/docker-compose.yml`

重点核对：

- [ ] API 端口映射为 `5001:8080`
- [ ] `ASPNETCORE_ENVIRONMENT=Development`
- [ ] 使用的是 `Database__ConnectionString`
- [ ] PostgreSQL 数据库名为 `movtools_server`
- [ ] PostgreSQL 用户为 `postgres`
- [ ] PostgreSQL 密码来自 `${DB_PASSWORD}`
- [ ] API 容器名是 `movtools-api`
- [ ] 数据库容器名是 `movtools-postgres`
- [ ] 数据卷路径是 `./postgres-data:/var/lib/postgresql/data`

如果这些项目对不上，请先不要启动容器，先修正配置。

---

## 4. Windows 部署前检查清单

- [ ] 已安装 Docker Desktop
- [ ] Docker Desktop 已正常启动
- [ ] Windows 已启用虚拟化
- [ ] WSL 2 已安装完成
- [ ] PowerShell 执行 `docker --version` 有输出
- [ ] PowerShell 执行 `docker compose version` 有输出
- [ ] 已把 `movtools-server` 复制到部署目录，例如 `D:\deploy\movtools-server`
- [ ] 当前目录下已存在 `.env`

---

## 5. Windows 首次启动清单

在 `movtools-server` 目录执行：

```powershell
docker compose up -d --build
```

执行后检查：

- [ ] 命令已执行完成，没有立即报错退出
- [ ] 第一次构建镜像时已等待下载完成
- [ ] 执行 `docker compose ps` 能看到 `movtools-api`
- [ ] 执行 `docker compose ps` 能看到 `movtools-postgres`
- [ ] 两个容器状态都是 `running` 或类似“已运行”状态

---

## 6. Windows 联网检查清单

- [ ] 已在本机浏览器打开 `http://localhost:5001/health`
- [ ] 本机访问返回成功
- [ ] 已执行 `ipconfig` 找到本机 IPv4 地址
- [ ] 已记录服务器地址，例如 `http://192.168.1.88:5001`
- [ ] 已在 Windows 防火墙中放行 TCP `5001`
- [ ] 其他局域网设备能访问 `http://服务器IP:5001/health`

---

## 7. Windows 业务验证清单

- [ ] 已测试登录接口 `http://服务器IP:5001/api/auth/login`
- [ ] 已确认默认账号 `admin / admin123` 可用，或已确认你自己的账号可用
- [ ] 第一台客户端已能正常连接服务器
- [ ] 第二台客户端已能正常连接服务器
- [ ] 多台设备登录后未出现明显连接失败
- [ ] SignalR 相关功能已做基础验证

---

## 8. 群辉 NAS 部署前检查清单

- [ ] DSM 已安装 `Container Manager`
- [ ] NAS 有足够可用存储空间
- [ ] NAS 可以联网拉取镜像
- [ ] 已在共享文件夹中创建部署目录，例如 `/volume1/docker/movtools-server`
- [ ] 已把 `movtools-server` 全部文件上传到该目录
- [ ] 已确认目录中存在 `src`
- [ ] 已确认目录中存在 `docker-compose.yml`
- [ ] 已确认目录中存在 `.env`

---

## 9. 群辉 NAS 启动清单

- [ ] 已打开 `Container Manager`
- [ ] 已进入“项目 / Project”
- [ ] 已选择“通过 docker-compose.yml 创建项目”
- [ ] 已选中 `/volume1/docker/movtools-server/docker-compose.yml`
- [ ] 已成功创建项目 `movtools-server`
- [ ] 已等待镜像下载和构建完成
- [ ] 已看到 `movtools-api` 容器运行中
- [ ] 已看到 `movtools-postgres` 容器运行中

---

## 10. 群辉 NAS 联网检查清单

- [ ] 已确认 NAS 的局域网 IP
- [ ] 已在浏览器打开 `http://NAS-IP:5001/health`
- [ ] 已在 DSM 防火墙中放行 TCP `5001`
- [ ] 其他设备能访问 `http://NAS-IP:5001/health`
- [ ] 客户端已改成连接 `http://NAS-IP:5001`

---

## 11. 启动失败排查清单

如果 `docker compose up -d --build` 或群辉项目启动失败，请按顺序检查：

- [ ] Docker Desktop 或 Container Manager 本身是否正常运行
- [ ] `.env` 是否存在
- [ ] `.env` 文件名是否正确
- [ ] `DB_PASSWORD` 是否为空
- [ ] `JWT_SIGNING_KEY` 是否为空
- [ ] `JWT_SIGNING_KEY` 是否至少 32 个字符
- [ ] `docker-compose.yml` 是否使用了当前仓库中的新版配置
- [ ] 是否误把端口改成了别的值
- [ ] 5001 端口是否被其他程序占用
- [ ] PostgreSQL 容器是否先启动成功

日志排查建议：

- Windows 看 `docker compose logs -f api`
- Windows 看 `docker compose logs -f postgres`
- 群辉看 `movtools-api` 容器日志
- 群辉看 `movtools-postgres` 容器日志

---

## 12. 连接失败排查清单

如果客户端打不开服务器或无法登录，请检查：

- [ ] 客户端填的是 `服务器IP`，不是 `localhost`
- [ ] 服务器本机 `http://localhost:5001/health` 可以打开
- [ ] 局域网设备访问 `http://服务器IP:5001/health` 可以打开
- [ ] Windows 或 DSM 防火墙已放行 `5001`
- [ ] API 容器仍在运行中
- [ ] 数据库容器仍在运行中
- [ ] 登录接口路径使用的是 `/api/auth/login`

---

## 13. 数据重置前确认清单

如果你准备重置数据库，先确认：

- [ ] 当前数据确实可以删除
- [ ] 不需要保留现有测试数据
- [ ] 已知道重置后默认账号会重新生成

Windows 重置命令：

```powershell
docker compose down -v
docker compose up -d --build
```

这会删除旧数据库数据，请谨慎操作。

---

## 14. 建议你实际执行时的最短路线

如果你只想先把测试服跑起来，按下面顺序走：

- [ ] 复制 `movtools-server/.env.example` 为 `movtools-server/.env`
- [ ] 修改 `.env` 中的两个值
- [ ] 执行 `docker compose up -d --build`
- [ ] 打开 `http://localhost:5001/health`
- [ ] 打开 `http://服务器IP:5001/health`
- [ ] 客户端改连 `http://服务器IP:5001`
- [ ] 多设备登录测试
