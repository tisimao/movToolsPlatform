## 2. 本批次验收目标

只有当以下目标同时成立，才可判定第十七批通过：

1. 镜头主状态与镜头二级状态已在产品和协议层彻底分层
2. 镜头二级状态只绑定当前镜头版本，版本升级后二级状态会自动重置
3. 制片、导演、制作人员三角色的二级状态动作边界与页面入口正确
4. 导演审片任务、审片工作台、反馈卡片保存与镜头详情回看形成闭环
5. 当前镜头二级状态未到 `DIRECTOR_APPROVED` 时，正式提交在客户端和服务端双层收口
6. 协议字段、接口路径、状态语义、权限约束与文档完全一致

---

## 3. 已固化业务边界（必须遵守）

1. 镜头主状态面向客户正式流程；镜头二级状态面向内部导演审片
2. 镜头二级状态稳定集合为：`NOT_IN_REVIEW`、`READY_FOR_REVIEW`、`IN_DIRECTOR_REVIEW`、`PENDING_FEEDBACK_FIX`、`FIX_UPDATED`、`DIRECTOR_APPROVED`
3. 当前镜头版本变化后，镜头二级状态必须自动重置为 `NOT_IN_REVIEW`
4. 制片负责组织性推进，导演负责反馈与内部通过，制作人员只负责 `FIX_UPDATED`
5. 当前二级状态不是 `DIRECTOR_APPROVED` 时，正式提交必须被拒绝
6. 导演反馈的最小产物是“标注图片 + 说明文字”的反馈卡片，且必须落服务器

---

## 4. 验收总原则

1. 先核对协议文档，再核对实现；若实现与文档不一致，以文档为准
2. 先验服务端权限与状态真相源，再验客户端界面行为
3. 不接受“按钮隐藏了但接口还能绕过”的假通过
4. 不接受“接口有了但页面不消费”或“页面做了但接口没兜底”的半通过
5. 不接受“旧版本二级状态残留到新版本”这类状态污染

## 6. 推荐验收顺序

1. 先读 `通信协议文档.md`，核对字段和接口
2. 再验服务端二级状态流转、提交拦截、版本升级重置
3. 再验客户端三角色页面和按钮收口
4. 再验导演审片闭环与反馈卡片回看
5. 最后做版本升级后状态重置与正式提交回归
