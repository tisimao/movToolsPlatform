# 审片功能二次迭代执行顺序总表

本文件用于统一说明“审片功能二次迭代”三份派工指令包的执行顺序、阶段目标与交付关系，便于直接派发给 `gpt-5.4-mini`。

- `D:\project\MovtoolsEx\Doc\通信协议文档.md`

## 2. 本轮问题背景

本轮二次迭代聚焦修复以下同类问题：

1. `上下文陪审镜头` 在导演工作台中被错误当作 `正式审片镜头`
2. 使用 `layout` 补位视频的镜头被错误展示正式审片操作
3. 导演工作台的 `审片完成` 与制片侧 `关闭任务` 语义混用
4. 任务完成判定、镜头操作能力、服务端状态推进三处口径不完全一致

---

## 3. 阶段划分

### 阶段一：协议与服务端状态机收口

执行文档：

1. `Doc/审片功能二次迭代后端派工指令包.md`

目标：

- 明确 `正式审片镜头 / 上下文陪审镜头` 的接口字段与兼容策略
- 修正导演 `complete` 与制片 `close` 的任务动作边界
- 在服务端补齐对 `context` 镜头的强约束，避免前端误判时写入脏状态

### 阶段二：导演工作台与制片任务页前端收口

执行文档：

2. `Doc/审片功能二次迭代前端派工指令包.md`

目标：

- 修正任务详情字段映射
- 确保 `context` 镜头只读、不可通过、不可写反馈
- 把导演 `审片完成` 改为调用正式 `complete` 接口
- 保持制片 `关闭任务` 仍走 `close`

### 阶段三：回归测试与验收固化

执行文档：

3. `Doc/审片功能二次迭代测试派工指令包.md`

目标：

- 覆盖后端接口测试、前端交互回归、自测清单
- 验证 `context` / `review`、`complete` / `close`、`layout` / 正式操作边界全部成立

---

## 4. 推荐执行顺序

1. 先执行后端派工包
2. 后端完成后执行前端派工包
3. 前后端都完成后执行测试派工包
4. 每阶段结束后都要回填文档变更、修改文件、自测结果与剩余风险

---

## 5. 派发要求

1. 一次只派发一份文档，不要把三份指令包混成一句话同时执行
2. 执行方必须先阅读 `项目固化文档.md` 和 `通信协议文档.md`
3. 执行方不得跳过测试，不得跳过文档更新
4. 若实现中发现协议与现状不一致，必须以本轮文档为准同步修正代码与测试
