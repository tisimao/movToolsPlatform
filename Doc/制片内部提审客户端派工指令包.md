# 制片内部提审客户端派工指令包

## 1. 文档用途

本文件用于把“制片内部提审派工链路”的客户端部分单独拆包，直接派发给前端开发或其他 AI 执行。

本包只关注客户端落地，重点解决：

1. 制片如何把多个镜头装配成任务
2. 导演如何按任务而不是按散镜头进入审片
3. 制片页和导演页如何对齐任务、镜头、二级状态口径
4. 现有单镜头 `进入审片` 按钮如何退居辅助或被任务派工替代

## 2. 本批次目标

这一批完成后，客户端必须具备：

1. 制片可在镜头页筛选并批量选择待提审镜头
2. 制片可在审片任务管理页创建草稿任务、添加镜头、调整顺序、指定提审版本、提交任务
3. 镜头页中的“进入审片”不再是派工主链路，主链路变为“加入任务并提交”
4. 导演仪表盘任务列表展示正式任务摘要，而不是单镜头拼装感
5. 导演审片页顶部展示任务镜头队列，支持按制片顺序切镜头
6. 导演保存反馈、继续审片、回跳最近反馈时都保留任务上下文
7. 镜头详情可查看所属任务并从镜头定位回任务

这一批不做：

1. 不重写底层播放器能力
2. 不重做现有全部导航体系之外的角色
3. 不用页面层假数据伪装任务能力
4. 不只改文案而不接真实接口

---

## 3. 已固化业务边界（必须遵守）

1. 制片派工主链路必须是“任务”，不是“单镜头改成审片中”
2. 导演主链路必须是“仪表盘 -> 任务 -> 审片工作台”
3. `待提审` 不等于导演可见
4. 导演看到任务的前提是服务端已存在正式 `ReviewTask`
5. 镜头页仍可展示二级状态，但任务入口必须优先落在任务页与任务卡片

---

## 4. 客户端问题定性

当前客户端需要按以下结论理解：

1. 制片镜头页当前“进入审片”按钮只是改镜头二级状态，不能代表已派工
2. 导演仪表盘虽然展示任务，但当前消费结构更像单镜头任务
3. 导演审片页目前缺少明确的“任务镜头队列”主结构
4. 制片端缺少独立的任务装配台，无法真正组织多镜头任务

## 5. 本批必须完成的客户端能力

### 5.1 制片镜头页改造

必须完成：

1. 保留二级状态操作：
   - `标记待提审`
   - `重新提审`
2. 将 `进入审片` 从主链路中降级，避免误导为“已派工”
3. 增加多选框和批量操作区
4. 增加批量动作：
   - 加入新任务
   - 加入草稿任务
   - 从草稿任务移除
5. 增加筛选项：
   - 二级状态
   - 是否已加入未关闭任务
   - 是否有导演反馈
   - 是否有可播放素材
6. 列表补充字段：
   - 所属任务
   - 最近提审版本
   - 最近导演反馈时间
   - 待处理反馈数量

### 5.2 制片审片任务管理页

必须新增独立页面，或在现有 `审片` 页中重构为制片任务管理页。

页面至少包含：

1. 任务列表
2. 任务详情 / 编辑区
3. 待提审镜头池
4. 任务进度区

必须支持：

1. 创建草稿任务
2. 编辑草稿任务
3. 选择目标导演
4. 填写任务名称、说明、截止时间
5. 添加多个镜头
6. 为每个镜头指定提审版本
7. 调整镜头顺序
8. 保存草稿
9. 提交任务
10. 关闭任务

当前正式客户端接口口径补充：

1. `POST /api/review-tasks/draft`
2. `GET /api/review-tasks/producer`
3. `GET /api/review-tasks/{id}/detail`
4. `PUT /api/review-tasks/{id}`
5. `POST /api/review-tasks/{id}/shots`
6. `DELETE /api/review-tasks/{id}/shots/{taskShotId}`
7. `PUT /api/review-tasks/{id}/shots/reorder`
8. `POST /api/review-tasks/{id}/submit`
9. `POST /api/review-tasks/{id}/start`
10. `POST /api/review-tasks/{id}/close`

当前状态投影补充：

1. 制片页显示的 `待提交` 是服务端 `ready` 的页面投影
2. 导演页显示的 `通过 / 返修` 分别消费服务端 `completed / closed`

### 5.3 导演仪表盘改造

必须确保任务卡片按正式任务展示，而不是按单镜头凑数：

1. 展示任务名称
2. 展示镜头数量
3. 展示提交人
4. 展示目标导演
5. 展示截止时间
6. 展示最近更新时间
7. 支持开始审片 / 继续审片

### 5.4 导演审片页改造

必须从“单镜头详情页”升级为“任务镜头队列工作台”：

1. 顶部展示任务基础信息
2. 顶部展示横向或网格镜头队列
3. 每个镜头项展示：
   - 顺序号
   - 镜头号
   - 提审版本
   - 实际播放版本
   - 反馈数量
   - 当前二级状态
4. 点击镜头项切换当前镜头
5. 显示任务整体进度
6. 保存反馈后刷新当前镜头与任务队列状态
7. 从仪表盘进入后自动定位任务与默认镜头

### 5.5 镜头详情回挂任务

必须支持：

1. 查看最近任务或所属任务列表
2. 一键定位回任务
3. 反馈卡片上显示所属任务
