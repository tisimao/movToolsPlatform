# 审片功能第一阶段客户端派工指令包

## 1. 文档用途

本文件用于把“审片任务可见性收口”这一阶段的客户端工作直接派发给其他 AI 或开发执行。

目标目录：`D:\project\MovtoolsEx\movtools-client\`

参考文档：

- `D:\project\MovtoolsEx\Doc\项目固化文档.md`
- `D:\project\MovtoolsEx\Doc\通信协议文档.md`
- `D:\project\MovtoolsEx\Doc\导演账号功能设计文档.md`
- `D:\project\MovtoolsEx\Doc\审片功能第一阶段服务器派工指令包.md`

---

## 2. 本阶段目标

在同一个客户端内，先把导演视图和制片视图对“未提交任务”的表现收口到正确语义。

本阶段完成后必须满足：

1. 制片仍可看到草稿/待提交任务
2. 导演页面不再展示未提交任务
3. 客户端不再假设“任务被创建”就等于“导演已收到任务”

---

## 3. 工作边界

1. 本阶段只改页面取数与展示语义，不新增“审片完成”按钮
2. 本阶段不做折叠归档列表
3. 本阶段以前后端联合结果为准，前端只做配套收口，不做越权兜底放宽

---

## 4. 推荐重点文件

- `src/pages/ReviewPage.tsx`
- `src/pages/ProducerTaskPage.tsx`
- `src/repositories/remote/reviewRepository.ts`
- 与任务状态映射、任务筛选、页面入口提示有关的 types / helpers

---

## 5. 必做事项

1. 调整导演审片页的任务列表心智，明确“这里只有制片已正式提交给导演的任务”
2. 若当前页面文案把 `ready`、`pending`、`in-review` 混成“都可审”，请按第一阶段服务端语义收口
3. 制片任务页保留草稿/待提交任务的查看与编辑入口
4. 若当前存在从镜头页或其他页面跳入导演审片的快捷入口，需避免把未提交任务误导航到导演工作台
5. 自测导演/制片两个角色切换后的任务列表表现

---

## 6. 禁止事项

1. 不要在客户端自行伪造“可见任务”结果覆盖服务端真实返回
2. 不要在本阶段顺手加入“审片完成”“关闭后归档”交互
3. 不要破坏制片现有的草稿编辑路径

---

## 7. 验收标准

1. 制片创建任务后仍能在制片页面看到该任务
2. 同一任务在未提交前，导演页面完全看不到
3. 提交后刷新导演页面，导演可以看到该任务
4. 导演页面文案与状态筛选不再误导为“创建即派发”
5. `npm run typecheck` 与 `npm run build` 通过

---
