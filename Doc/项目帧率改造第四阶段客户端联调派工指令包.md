# 项目帧率改造第四阶段客户端联调派工指令包

## 1. 文档用途

本文件用于把“客户端正式消费服务端项目 fps 配置并完成联调回归”这一阶段的工作直接派发给其他 AI 或开发执行。

目标目录：`D:\project\MovtoolsEx\movtools-client\`

参考文档：

- `D:\project\MovtoolsEx\Doc\项目固化文档.md`
- `D:\project\MovtoolsEx\Doc\通信协议文档.md`
- `D:\project\MovtoolsEx\Doc\项目帧率改造第一阶段客户端派工指令包.md`
- `D:\project\MovtoolsEx\Doc\项目帧率改造第二阶段客户端派工指令包.md`
- `D:\project\MovtoolsEx\Doc\项目帧率改造第三阶段服务端派工指令包.md`

---

## 2. 本阶段目标

本阶段只解决“客户端从服务端读取项目 fps 并在导演审片页、镜头详情页、项目相关展示中统一消费”的问题。

完成后必须满足：

1. 激活项目后，客户端能读取该项目正式 fps
2. 导演审片页按项目 fps 计算帧号、步进、seek
3. 镜头详情页导演反馈回放按项目 fps 逐帧解算
4. 没有项目 fps 时仍回退到默认 30fps

---

## 3. 工作边界

1. 本阶段以联调消费为主，不负责新增服务端字段
2. 本阶段不修改导演审片绘制、草稿、提交主流程
3. 本阶段允许调整 store / helper / 映射层，但要避免打散统一入口

---

## 4. 推荐重点文件

- `src/pages/ReviewPage.tsx`
- `src/pages/LensPage.tsx`
- `src/components/lens/DirectorFeedbackPlayback.tsx`
- 项目/工作区相关 store
- 项目/远程仓储映射层
- 如有必要：统一 fps helper

---

## 5. 必做事项

1. 打通项目 fps 从服务端响应到前端 store 的链路
2. 用项目 fps 替换回放页和镜头详情页中的默认 fps 读取
3. 保留 `30fps` 默认兜底，避免旧项目或旧数据异常
4. 回归验证多种场景：新项目 30fps、自定义 fps、旧项目无 fps 回退 30
5. 交付中明确哪些页面已经正式消费项目 fps

---

## 6. 禁止事项

1. 禁止修改 `ReviewPage.tsx` 导演审片主编辑流程语义
2. 禁止出现读取项目 fps 一份、镜头详情回放又读另一份的双真相源
3. 禁止删除默认 30fps 回退分支
4. 禁止漏改镜头详情页与导演审片页中的任一侧

---

## 7. 验收标准

1. 客户端可读取并使用项目 fps
2. 导演审片页与镜头详情页导演反馈回放在同一项目下 fps 一致
3. 无 fps 旧项目仍稳定按 30fps 运行
4. `npm run typecheck` 与 `npm run build` 通过

---
