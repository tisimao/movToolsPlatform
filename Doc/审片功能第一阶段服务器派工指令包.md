# 审片功能第一阶段服务器派工指令包

## 1. 文档用途

本文件用于把“审片任务可见性收口”这一阶段的服务端工作直接派发给其他 AI 或开发执行。

目标目录：`D:\project\MovtoolsEx\movtools-server\`

参考文档：

- `D:\project\MovtoolsEx\Doc\项目固化文档.md`
- `D:\project\MovtoolsEx\Doc\通信协议文档.md`
- `D:\project\MovtoolsEx\Doc\导演账号功能设计文档.md`
- `D:\project\MovtoolsEx\Doc\审片功能第一阶段客户端派工指令包.md`

---

## 2. 本阶段目标

只先解决一个问题：制片未执行“提交审片”前，导演账号不得看见任务。

本阶段完成后必须满足：

1. 草稿任务只属于制片装配态
2. 导演任务列表不返回未提交任务
3. 导演任务详情接口也不能绕过列表直接读到未提交任务
4. 制片提交后，导演才获得该任务可见性

---

## 3. 工作边界

1. 本阶段只改任务可见性与读取权限，不改“审片完成”“关闭任务”规则
2. 本阶段不引入新的复杂状态机，只收口现有 `draft / ready / pending / in-review / completed / closed` 的可见性语义
3. 本阶段不处理 UI 折叠列表，只保证服务端真相源正确

---

## 4. 推荐重点文件

- `src/Movtools.Server.Infrastructure/Services/ReviewService.cs`
- `src/Movtools.Server.Api/Controllers/ReviewsController.cs`
- `src/Movtools.Server.Domain/Entities/ReviewTask.cs`
- 与任务列表查询、任务详情权限、状态过滤相关的 service / mapper / tests

---

## 5. 必做事项

1. 明确服务端“导演可见任务”的正式前提：任务已经由制片执行 `提交审片`，且任务尚未关闭
2. 收口 `GET /api/review-tasks` 或等价导演任务列表接口，不再把 `ready` 或其他未正式提交任务返回给导演
3. 收口导演读取任务详情权限；若任务未提交，则导演即使知道 `taskId` 也不得读到任务详情
4. 保留制片读取草稿、待提交任务、已提交任务的能力
5. 若当前实现把 `ready` 也视为导演可见，请重新定义其语义：`ready` 仍属于制片侧装配完成但尚未提交给导演的状态
6. 增加最小测试覆盖：草稿创建后导演不可见，提交后导演可见

---

## 6. 禁止事项

1. 不要只在客户端隐藏任务，必须由服务端做最终过滤
2. 不要让导演详情接口比导演列表接口更宽松
3. 不要顺手改“完成/关闭”链路，本阶段只收口可见性

---

## 7. 验收标准

1. 制片创建草稿任务后，导演列表中看不到该任务
2. 制片将任务提交后，导演列表中可以看到该任务
3. 导演无法通过直接请求详情接口读取未提交任务
4. 现有制片任务管理功能不被破坏
5. 服务端测试通过
