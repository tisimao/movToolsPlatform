# 项目帧率改造第三阶段服务端派工指令包

## 1. 文档用途

本文件用于把“项目 fps 服务端持久化与回读”这一阶段的服务端工作直接派发给其他 AI 或开发执行。

目标目录：`D:\project\MovtoolsEx\movtools-server\`

参考文档：

- `D:\project\MovtoolsEx\Doc\项目固化文档.md`
- `D:\project\MovtoolsEx\Doc\通信协议文档.md`
- `D:\project\MovtoolsEx\Doc\项目帧率改造第二阶段客户端派工指令包.md`

---

## 2. 本阶段目标

本阶段只解决“项目默认 fps 进入服务端正式协议、数据库持久化、查询回读”的问题。

完成后必须满足：

1. 项目创建请求支持 fps 字段
2. 项目实体正式持久化 fps
3. 项目详情/工作区/相关查询可回读 fps
4. 未显式填写 fps 时，服务端默认值为 `30`

---

## 3. 工作边界

1. 本阶段只改服务端
2. 本阶段不负责客户端页面改造
3. 本阶段必须同步更新协议文档，不能只改代码
4. 本阶段需要补数据库迁移与必要测试

---

## 4. 推荐重点文件

- 项目创建相关 Contract / DTO
- Project 实体与配置
- Project 查询/映射相关服务
- EF migration / snapshot
- 相关单元测试或 API 测试

---

## 5. 必做事项

1. 给项目创建请求 DTO 增加 fps 字段
2. 给项目实体增加 fps 持久化字段，默认值设为 `30`
3. 更新项目列表、项目详情、工作区或任何客户端依赖的回读 DTO
4. 补 migration，确保开发环境可自动迁移
5. 更新测试，覆盖“默认 30 / 显式 30 / 显式其他合法值”场景

---

## 6. 禁止事项

1. 禁止只改创建请求，不改查询回读
2. 禁止只改代码，不改 `通信协议文档.md`
3. 禁止让数据库默认值、实体默认值、接口默认值三者不一致
4. 禁止擅自改动导演审片反馈协议

---

## 7. 验收标准

1. 服务端项目创建协议正式支持 fps
2. 项目 fps 能稳定落库并回读
3. 未传 fps 时默认值为 `30`
4. 相关测试通过

---
