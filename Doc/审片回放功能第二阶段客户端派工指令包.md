# 审片回放功能第二阶段客户端派工指令包

## 1. 文档用途

本文件用于把“镜头详情页导演反馈回放组件逐帧化”这一阶段的客户端工作直接派发给其他 AI 或开发执行。

目标目录：`D:\project\MovtoolsEx\movtools-client\`

参考文档：

- `D:\project\MovtoolsEx\Doc\项目固化文档.md`
- `D:\project\MovtoolsEx\Doc\通信协议文档.md`
- `D:\project\MovtoolsEx\Doc\镜头详情页设计文档.md`
- `D:\project\MovtoolsEx\Doc\审片回放功能第一阶段客户端派工指令包.md`

---

## 2. 本阶段目标

本阶段只解决“镜头详情页导演反馈视频回放按当前帧解算绘制”的问题，不改导演审片主编辑流程。

完成后必须满足：

1. 镜头详情页导演反馈回放不再一次性渲染整轮最终遮罩
2. 回放组件会根据当前视频播放帧动态解析可见绘制内容
3. 点击反馈卡片定位到目标帧后，视频画面上展示该帧应看到的绘制状态
4. 导演审片页现有绘制/暂停标注/草稿/提交流程保持不变

---

## 3. 工作边界

1. 本阶段只重构镜头详情页的 `DirectorFeedbackPlayback` 或其等价回放组件
2. 本阶段不修改导演审片工作台的编辑交互
3. 本阶段不改服务端协议
4. 本阶段允许复用现有绘制解析 helper，但不得改变其在导演审片页中的既有语义

---

## 4. 推荐重点文件

- `src/components/lens/DirectorFeedbackPlayback.tsx`
- `src/lib/reviewDrawingResolver.ts`
- `src/components/AnnotationCanvas.tsx`
- 必要时配套样式文件

---

## 5. 必做事项

1. 梳理当前 `DirectorFeedbackPlayback` 是如何计算遮罩的，明确为什么它只显示最终态而不是逐帧态
2. 建立“当前视频时间 -> 当前帧号 -> 当前可见路径”的回放链路
3. 在不影响视频 seek 的前提下，让点击反馈卡片后的帧定位与绘制显示保持一致
4. 确保不同视频比例下，绘制层与视频显示区域对齐
5. 若现有实现使用静态图片叠层无法满足逐帧回放，可替换为 canvas/annotation 层，但只限镜头详情页回放组件内部

---

## 6. 禁止事项

1. 禁止修改 `src/pages/ReviewPage.tsx` 的绘制流程
2. 禁止修改导演审片提交接口、草稿保存接口、反馈生成协议
3. 禁止继续用 `Number.MAX_SAFE_INTEGER` 一次性解算整轮最终态冒充回放态
4. 禁止为了实现逐帧回放而删减镜头详情页现有点击定位能力
5. 禁止在本阶段额外改造镜头详情页其他分页

---

## 7. 验收标准

1. 镜头详情页导演反馈视频回放时，绘制内容会随着视频当前帧变化而变化
2. 点击某条反馈卡片后，视频 seek 到对应帧，画面显示该帧应有的导演绘制
3. 不同视频尺寸下，绘制层与视频主体不明显错位
4. `ReviewPage.tsx` 不发生修改，或仅允许零业务语义的 import/type 调整
5. `npm run typecheck` 与 `npm run build` 通过
