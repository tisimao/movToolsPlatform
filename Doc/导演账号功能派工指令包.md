## 2. 当前拆分原则

1. 客户端派工包负责页面结构、播放器交互、绘制本地数据、反馈草稿、镜头详情遮罩播放与最终联调验证
2. 服务端派工包负责协议、落库、镜头级绘制数据提交/回读、反馈轮次聚合、镜头状态更新、镜头详情投影与测试
3. 两侧都必须按“阶段完成可运行、阶段之间可联调”的原则执行，不允许只写最终大而全目标而没有中间可验证里程碑
4. 若任一阶段发现与 `导演账号功能设计文档.md` 不一致，必须先更新协议/固化文档后再改实现

## 3. 阶段对齐要求

导演账号功能最终必须同时满足以下结果：

1. 导演主入口固定为 `仪表盘 -> 审片任务 -> 审片工作台`
2. 审片工作台按任务镜头顺序连播，镜头队列颜色与反馈状态一致
3. 暂停态可绘制，绘制本地保存，切镜头/刷新/关闭后不丢失
4. 绘制底层按单帧存储，显示层按继承链解释，`清屏` 会打断前序继承
5. 反馈说明与绘制弱绑定，允许只提文字，不允许只提绘制
6. 点击 `提交反馈` 时，上传当前镜头本轮全部文字反馈、全部本地绘制数据与镜头状态
7. 点击 `镜头通过` 时，只上传镜头通过状态
8. 镜头详情页 `导演反馈` 分页支持“显示导演反馈遮罩”开关，且只显示当前轮正式提交内容
9. 当前轮提交成功后，本地反馈内容与本地绘制内容清空，进入下一轮空白编辑态

## 4. 执行顺序建议

建议按以下顺序派发和联调：

1. 先执行服务端第 1、2 阶段，固化反馈轮次与绘制提交/回读协议
2. 再执行客户端第 1、2 阶段，收口导演入口、工作台布局与镜头状态交互
3. 然后并行执行客户端第 3、4 阶段与服务端第 3 阶段，完成绘制本地模型、提交链路、镜头详情遮罩能力
4. 最后执行客户端第 5 阶段与服务端第 4 阶段，完成联调、测试与验收收口
