# 审片功能第四阶段客户端派工指令包

## 1. 文档用途

本文件用于把“制作人员反馈消费与修改完成交互”这一阶段的客户端工作直接派发给其他 AI 或开发执行。

目标目录：`D:\project\MovtoolsEx\movtools-client\`

参考文档：

- `D:\project\MovtoolsEx\Doc\项目固化文档.md`
- `D:\project\MovtoolsEx\Doc\通信协议文档.md`
- `D:\project\MovtoolsEx\Doc\导演账号功能设计文档.md`
- `D:\project\MovtoolsEx\Doc\审片功能第四阶段服务器派工指令包.md`

---

## 2. 本阶段目标

让制作人员在同一个客户端的镜头详情页中消费导演反馈，并对返修镜头执行“修改完成”。

本阶段完成后必须满足：

1. 制作人员可在镜头详情页看到导演反馈内容、标注图、遮罩和版本信息
2. 对于 `PENDING_FEEDBACK_FIX` 镜头，制作人员可点击“修改完成”
3. 制作人员没有导演任务列表入口，也不进入导演工作台

---

## 3. 工作边界

1. 本阶段只改制作人员视图
2. 不新增任何任务级导航给 maker
3. 若导演反馈分页已存在，应优先复用，不要重做一套并行 UI

---

## 4. 推荐重点文件

- `src/pages/LensPage.tsx`
- `src/components/lens/DirectorFeedbackPlayback.tsx`
- `src/lib/directorFeedback.ts`
- `src/repositories/remote/lensRepository.ts`
- 与 maker 角色页面入口、按钮展示条件、反馈分页相关文件

---

## 5. 必做事项

1. 在制作人员可见的镜头详情中稳定展示导演反馈内容与回放能力
2. 对 `PENDING_FEEDBACK_FIX` 镜头展示“修改完成”操作，并调用正式二级状态接口
3. 对非 `PENDING_FEEDBACK_FIX` 镜头隐藏或禁用该操作，并给出正确提示
4. 确认制作人员导航和页面入口中没有任务级审片入口残留
5. 自测制作人员视角下的反馈阅读、遮罩回放和修改完成动作

---

## 6. 禁止事项

1. 不要给制作人员暴露导演任务列表或导演审片按钮
2. 不要把“修改完成”错误显示给非返修镜头
3. 不要为 maker 单独复制一套导演反馈展示逻辑，尽量复用既有详情/回放组件

---

## 7. 验收标准

1. 制作人员能在镜头详情读到导演反馈、图片和遮罩
2. `PENDING_FEEDBACK_FIX` 镜头可以执行“修改完成”
3. 非返修镜头不能错误显示该动作
4. 制作人员页面没有任务级审片入口
5. `npm run typecheck` 与 `npm run build` 通过

---
