# 审片功能二次迭代测试派工指令包

本文件用于把“审片功能二次迭代”的测试、回归与验收工作直接派发给其他 AI 或开发执行。

## 2. 本阶段目标

验证本轮二次迭代不会再出现“陪审镜头可正式操作”与“导演完成动作走错状态流转”的问题。

本阶段完成后必须满足：

1. `review/context` 参与类型在服务端与前端都能稳定回读
2. `context` 镜头不能写正式反馈、不能写通过结论、不能阻塞任务完成
3. 导演完成任务走 `complete`，制片关闭任务走 `close`
4. `layout` 播放源与 `context` 参与类型不再混用

---

## 3. 必测场景

### 场景 1：mixed task 主场景

1. 创建包含 6 个镜头的任务
2. 前 5 个镜头为 `review`
3. 第 6 个镜头为 `context`
4. 第 6 个镜头仅依赖 `layout` 视频补位
5. 验证导演端第 6 个镜头不可正式操作

### 场景 2：导演完成任务

1. 正式镜头中部分有反馈、部分已通过
2. `context` 镜头不处理正式结论
3. 验证导演端可完成任务
4. 验证任务状态进入 `completed`

### 场景 3：制片关闭任务

1. 未完成任务时 close 被拒绝
2. 已完成任务时 close 成功
3. close 后导演端列表中不再可见

### 场景 4：字段兼容

1. 验证任务详情回包参与类型字段可被前端稳定识别
2. 验证旧兼容路径不会导致 `context` 回落成 `review`

## 5. 必做事项

1. 新增或修正服务端集成测试，覆盖以下断言：
   - mixed review/context task 中 context 不被推进到正式状态机
   - context 镜头不能创建正式反馈
   - context 镜头不能写内部通过
   - task complete 只统计 review 镜头
   - producer close 只能作用于 completed task
2. 若前端具备测试能力，补充仓储映射或页面级测试，覆盖：
   - `participationMode` / `reviewParticipationMode` 映射
   - ReviewPage 中 context 按钮隐藏或禁用
   - complete 与 close 的接口调用分离
3. 至少执行一次人工回归：
   - 导演账号进入 mixed task
   - 切到第 6 个 context layout 镜头
   - 检查按钮、编辑器、完成按钮状态
4. 输出失败场景与剩余风险，特别是历史任务数据兼容风险

---

## 6. 禁止事项

1. 不要只验证 happy path
2. 不要只看 UI，要同时核对接口状态变化
3. 不要遗漏 `layout` 源但 `review` 身份、以及 `context` 身份但非 layout 源的交叉场景
4. 不要跳过文档核对

---

## 7. 验收标准

1. mixed task 回归通过
2. complete / close 语义验证通过
3. context 镜头正式写操作保护验证通过
4. 至少一组前端映射或交互测试已补齐，若当前基建不足则明确记录人工回归结果
5. 文档、代码、测试三者口径一致
