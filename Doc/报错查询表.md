# 报错查询表 - 快速定位问题指南

## 1. HTTP 状态码速查

| 状态码 | 含义 | 常见原因 | 排查方向 |
|--------|------|----------|----------|
| **0** | 网络连接失败 | 服务端未启动 / 端口不通 / CORS 阻塞 / API 基础地址配置错误 | 检查服务端是否运行，端口是否与启动配置一致（当前 VS 默认 5001） |
| **404** | 接口不存在 | 路径不匹配 / 路由拼写错误 / 服务端未启动 | 核对客户端 API 路径 vs 服务端路由 |
| **401** | 未授权 | Token 无效 / 已过期 / 未登录 | 检查登录是否成功，Token 是否有效 |
| **403** | 禁止访问 | 权限不足，角色不匹配 | 检查用户角色是否有权限调用该接口 |
| **500** | 服务器错误 | 服务端代码异常 | 查看服务端控制台/日志 |
| **502** | 网关错误 | 服务端进程崩溃 / 端口被占用 | 重启服务端，检查端口占用 |
| **503** | 服务不可用 | 服务端正在启动或维护中 | 等待服务端就绪 |

## 2. 常见 404 排查清单

### 排查步骤

```
1. 检查服务端是否运行
   → 浏览器访问 http://localhost:5001/health 是否返回 200

2. 检查端口是否匹配
   → 客户端 VITE_API_BASE_URL 是否为 http://localhost:5001

3. 检查服务端启动方式
   → 如果是 VS / Rider 启动，确认 launchSettings.json 的 applicationUrl
   → 当前默认值为 http://localhost:5001

4. 检查路径前缀是否正确
   → 客户端路径是否包含 /api 前缀
   → 服务端路由定义 [Route("api/[controller]")]

5. 检查 HTTP 方法是否匹配
   → 登录是 POST，查询是 GET

6. 检查 Controller 和 Action 名称拼写
   → AuthController vs authController
```

### 客户端提示“无法连接到服务端，请检查 API 地址或服务端状态”时

| 检查项 | 说明 | 修正方法 |
|--------|------|----------|
| API 基础地址 | 客户端默认读 `VITE_API_BASE_URL` | 将 `.env.local` 设为 `http://localhost:5001`，或按实际启动端口修改 |
| 服务端监听端口 | VS / Rider 默认启动端口 | 查看 `movtools-server/src/Movtools.Server.Api/Properties/launchSettings.json` |
| 健康检查 | 客户端启动脚本会探测 `/health` | 确认 `http://localhost:5001/health` 可访问 |
| CORS | 端口变更后可能跨域 | 将客户端来源加入 `Server:AllowedOrigins` |

### 路径对照表

| 功能 | 客户端配置 | 服务端路由 | 完整路径 |
|------|-----------|-----------|----------|
| 登录 | `VITE_API_LOGIN_PATH=/api/auth/login` | `[Route("api/[controller]")]` + `[HttpPost("login")]` | `POST /api/auth/login` |
| 当前用户 | `VITE_API_ME_PATH=/api/auth/me` | `[Route("api/[controller]")]` + `[HttpGet("me")]` | `GET /api/auth/me` |
| 健康检查 | `VITE_API_HEALTH_PATH=/health` | `app.MapHealthChecks("/health")` | `GET /health` |

## 3. 认证相关错误

| 错误现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 登录返回 401 | 用户名或密码错误 | 确认默认账号 admin / Admin@123456 |
| 登录返回 404 | 路径错误 | 参考本文档第 2 节 |
| 登录成功但跳转后仍提示未登录 | Token 未正确保存 | 检查 localStorage 的 auth.session.v1 |
| 获取当前用户返回 401 | Token 已过期或无效 | 重新登录获取新 Token |
| 受保护接口返回 403 | 权限不足 | 检查用户角色是否有权限 |

## 6. 服务端启动检查

### 启动步骤

1. 确保 PostgreSQL 已启动（端口 5432）
2. 使用 VS/Rider 启动 movtools-server
3. 确认控制台输出 `Application started. Press Ctrl+C to shut down.`
4. 访问 `http://localhost:5001/health` 确认返回 `{"status":"Healthy"}`

### 常见启动错误

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| `Cannot connect to database` | PostgreSQL 未启动 | 启动 PostgreSQL 服务 |
| `relation "users" does not exist` | Migration 未执行 | 运行 `dotnet ef database update` |
| `Port 5001 is already in use` | VS 默认端口被占用 | 关闭占用进程，或修改 `launchSettings.json` / 启动端口 |

## 7. 客户端启动检查

### 启动步骤

1. 确保服务端已在运行
2. 执行 `start-dev-test.bat` 或 `npm run dev`
3. 客户端启动成功后访问 `http://localhost:5173`
4. 输入账号密码登录

### 常见启动错误

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| `Node version < 20` | Node 版本过低 | 升级到 Node 20+ |
| `Electron not found` | Electron 未安装 | 删除 node_modules 后重新 npm install |
| `Cannot connect to server` | 服务端未运行 / API_BASE_URL 配错 | 先启动服务端，并确认 `.env.local` 指向实际监听端口 |

## 8. 快速诊断命令

### 服务端健康检查

```powershell
# 手动检查服务端是否运行
Invoke-WebRequest -Uri 'http://localhost:5001/health' -UseBasicParsing
```

### 检查客户端实际连接地址

```powershell
# 查看客户端运行时默认 API 地址
Get-Content .\.env.local

# 如需临时验证，手动请求健康检查
Invoke-WebRequest -Uri 'http://localhost:5001/health' -UseBasicParsing
```

### 查看服务端所有可用路由

```csharp
// 在 Program.cs 添加以下代码后重启
app.MapGet("/debug/routes", () => 
    app.Services.GetRequiredService<Microsoft.AspNetCore.Routing.IEndpointRoutingEndpointResolver>().GetCandidateEndpoints());
```

或者直接查看控制器源码中的 `[Route(...)]` 和 `[Http...]` 属性。
