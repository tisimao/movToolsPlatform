# 审片功能第三阶段客户端派工指令包

## 1. 文档用途

本文件用于把“制片归档视图与导演已完成列表收口”这一阶段的客户端工作直接派发给其他 AI 或开发执行。

目标目录：`D:\project\MovtoolsEx\movtools-client\`

参考文档：

- `D:\project\MovtoolsEx\Doc\项目固化文档.md`
- `D:\project\MovtoolsEx\Doc\通信协议文档.md`
- `D:\project\MovtoolsEx\Doc\审片功能第三阶段服务器派工指令包.md`

---

## 2. 本阶段目标

让同一个客户端中的导演页和制片页都具备“主工作区 + 折叠归档区”的正确表现。

本阶段完成后必须满足：

1. 导演完成任务后，任务进入“已完成任务列表”并默认折叠
2. 制片关闭任务后，任务进入“已关闭审片任务列表”并默认折叠
3. 已完成/已关闭任务不再占用主操作区空间
4. 任务一旦被关闭，会从导演侧已完成列表移除

---

## 3. 工作边界

1. 本阶段主要改列表与视图组织，不新增制作人员功能
2. 归档区默认折叠，但仍需可展开回看
3. 若“删除关闭任务反馈”入口需要占位，可先做安全入口和提示，不强推物理删除实现

---

## 4. 推荐重点文件

- `src/pages/ReviewPage.tsx`
- `src/pages/ProducerTaskPage.tsx`
- `src/repositories/remote/reviewRepository.ts`
- 与任务列表分组、折叠状态、本地筛选文案相关的 store / helper / types

---

## 5. 必做事项

1. 导演页把主工作列表与“已完成任务列表”拆开，已完成列表默认折叠
2. 制片页把主工作列表与“已关闭审片任务列表”拆开，已关闭列表默认折叠
3. 导演页在任务被制片关闭后，自动从已完成列表移除该任务
4. 制片页在关闭任务后，把任务从主工作区移动到已关闭归档区
5. 为已关闭任务预留“手动删除反馈/清场”入口位，但不要让入口误触发危险动作

---

## 6. 禁止事项

1. 不要把折叠归档区做成默认展开占满页面
2. 不要让已关闭任务继续保留在导演可见区域
3. 不要把 completed 和 closed 在前端文案上混成同一种状态

---

## 7. 验收标准

1. 导演完成后的任务进入折叠的已完成区
2. 制片关闭后的任务进入折叠的已关闭区
3. 已完成/已关闭任务不再挤占主工作列表
4. 关闭任务后导演页面不再看见该任务
5. `npm run typecheck` 与 `npm run build` 通过

---

