# 项目帧率改造第一阶段客户端派工指令包

## 1. 文档用途

本文件用于把“客户端回放链路统一默认 30fps”这一阶段的工作直接派发给其他 AI 或开发执行。

目标目录：`D:\project\MovtoolsEx\movtools-client\`

参考文档：

- `D:\project\MovtoolsEx\Doc\项目固化文档.md`
- `D:\project\MovtoolsEx\Doc\通信协议文档.md`
- `D:\project\MovtoolsEx\Doc\镜头详情页设计文档.md`
- `D:\project\MovtoolsEx\Doc\审片回放功能第三阶段客户端派工指令包.md`

---

## 2. 本阶段目标

本阶段只解决“客户端所有回放/帧计算默认值从 24fps 收口为统一 30fps”的问题，不引入项目级可配置持久化。

完成后必须满足：

1. 客户端回放链路不再散落写死 `24`
2. 导演审片页与镜头详情页使用同一份默认 fps 入口
3. 当前没有项目级 fps 配置时，统一按 `30fps` 计算
4. 不改变 `ReviewPage.tsx` 的导演审片绘制、草稿、提交主流程

---

## 3. 工作边界

1. 本阶段只改客户端
2. 本阶段不改服务端协议、不改数据库
3. 本阶段不在项目创建页增加 fps 表单
4. 本阶段允许重构 fps helper，但不得改变导演审片业务语义

---

## 4. 推荐重点文件

- `src/pages/ReviewPage.tsx`
- `src/components/lens/DirectorFeedbackPlayback.tsx`
- `src/pages/LensPage.tsx`
- 如有必要新增：`src/lib/playbackFps.ts`

---

## 5. 必做事项

1. 搜索所有回放/帧定位/逐帧步进相关的 `24`、`fps` 硬编码
2. 抽出统一的默认 fps 读取入口，默认值改为 `30`
3. 让导演审片页与镜头详情页导演反馈回放共用这份 fps
4. 检查帧号计算、逐帧 seek、slider step、帧时长展示是否同步改到 30fps
5. 保证镜头详情页导演反馈回放与导演审片页在默认 fps 下继续一致

---

## 6. 禁止事项

1. 禁止修改导演审片的绘制、草稿、提交主流程
2. 禁止在本阶段新增服务端字段或数据库迁移
3. 禁止让 `ReviewPage.tsx`、`LensPage.tsx`、回放组件各自维护不同 fps 默认值
4. 禁止继续保留分散的 `24fps` 硬编码入口

---

## 7. 验收标准

1. 当前项目无额外配置时，导演审片页按 `30fps` 计算帧号与逐帧步进
2. 镜头详情页导演反馈回放按 `30fps` 推算当前帧与 seek 时间
3. 导演审片页与镜头详情页不再出现一边 24fps、一边 30fps 的分叉
4. `npm run typecheck` 与 `npm run build` 通过

---
