## 2. 本批次验收目标

只有当以下目标同时成立，才可判定本批通过：

1. 制片未提交前，导演绝对看不到该审片任务
2. 导演只能对已正式派发任务执行审片，并且必须在全部正式审片镜头都已处理后才能点击 `审片完成`
3. 制片只能关闭已完成导演审片的任务
4. 导演已完成列表与制片已关闭列表都进入折叠归档区，不再占主操作界面
5. 任务一旦关闭，导演侧彻底不可见
6. 制作人员能在镜头详情查看导演反馈，并能将返修镜头推进到 `FIX_UPDATED`

---

## 3. 已固化业务边界（必须遵守）

1. 草稿任务只属于制片装配态，不属于导演待办
2. 导演可见任务的前提是：制片已执行 `提交审片`，且任务尚未被制片关闭
3. `正式审片镜头` 与 `上下文陪审镜头` 必须严格区分
4. `上下文陪审镜头` 不参与通过、反馈、完成判定与待处理统计
5. 导演任务完成不等于制片任务关闭
6. 关闭任务默认保留反馈事实，直到制片后续手动清场
7. 制作人员只消费镜头级反馈，不消费任务级导演工作台

---

## 4. 验收总原则

1. 先核对文档口径，再核对服务端真相源，最后核对客户端角色页面
2. 不接受“前端隐藏了，但接口仍能读到”的假通过
3. 不接受“导演已完成”和“制片已关闭”语义混用的半通过
4. 不接受“context 镜头阻塞任务完成”的错误实现
5. 不接受“制作人员能看见导演任务列表”或“能越权操作其他镜头反馈”的权限泄漏
