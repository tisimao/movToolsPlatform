# 制作人员账号功能设计文档

## 1. 文档目的

本文件用于固化制作人员账号在导演审片闭环中的职责、页面能力、反馈消费方式与权限边界，确保后续导演功能增强时，制作人员端功能保持一致，不发生角色跑偏。

## 2. 角色定位

制作人员账号的核心目标不是管理任务，而是处理分配给自己的镜头，并准确接收导演反馈。

围绕导演审片新增能力后，制作人员的核心职责变为：

1. 查看自己参与项目中的镜头
2. 查看镜头当前版本与可播放素材
3. 在镜头详情中查看导演反馈
4. 根据反馈更新镜头版本
5. 处理镜头内部审片二级状态中的“已按反馈修改”动作
6. 等待制片组织下一轮提审

## 3. 左侧主导航设计

制作人员账号左侧主导航继续收口为：

1. 仪表盘
2. 镜头
3. 设置

导演新增功能不应破坏制作人员端的轻量导航结构。

## 4. 制作人员与导演流程关系

制作人员不直接创建导演反馈，但必须高效消费导演反馈。

协同关系建议固化如下：

1. 制片整理任务并提交导演
2. 导演在审片工作台保存反馈卡片
3. 反馈自动挂到对应镜头
4. 制作人员在镜头详情中查看反馈图片与说明
5. 制作人员完成修改并产出新版本
6. 制作人员将镜头标记为 `已按反馈修改`

## 5. 制作人员仪表盘设计关注点

制作人员仪表盘建议围绕“我需要处理什么镜头”展示。

建议区块：

1. 我可访问的项目
2. 我的镜头概览
3. 待处理反馈提醒
4. 最近更新镜头

### 建议新增的反馈提醒字段

1. 有新导演反馈的镜头数
2. 需修改镜头数
3. 最近反馈更新时间
4. 快速进入镜头详情按钮

## 6. 镜头页面设计

制作人员端的“镜头”页是处理导演反馈的主入口。

### 6.1 列表字段建议

1. 镜头号
2. 所属项目 / 集
3. 当前最新版本
4. 当前镜头主状态
5. 当前内部审片二级状态
6. 最近导演反馈时间
7. 导演反馈数量
8. 是否存在未处理反馈

### 6.2 筛选建议

1. 项目
2. 制作人员本人
3. 主状态
4. 二级状态
5. 是否有导演反馈
6. 是否需修改
7. 版本号区间

### 6.3 列表关键操作

1. 查看镜头详情
2. 查看版本信息
3. 查看导演反馈
4. 标记已按反馈修改
5. 刷新本机文件匹配

制作人员的“标记已按反馈修改”按钮建议遵循以下规则：

1. 仅在当前镜头二级状态为 `待处理反馈` 时显示
2. 仅镜头负责人本人可操作
3. 点击后将二级状态更新为 `已按反馈修改`
4. 该动作不等于重新提审，也不等于内部通过

## 7. 镜头详情页中的导演反馈设计

这是制作人员端本轮最重要的新增区块。

### 7.1 页面定位

镜头详情页需要成为制作人员查看导演反馈的稳定入口。

补充口径：

1. 镜头详情页中的 `导演反馈` 与 `镜头反馈` 是同级分页
2. `导演反馈` 用于查看导演反馈卡片
3. `镜头反馈` 通过 `记录` 分页查看镜头主状态链路中的返修记录与返修图片

### 7.2 新增分页名称建议

`导演反馈`

### 7.3 展示方式

导演反馈建议采用反馈卡片流展示，每条卡片都对应导演在某一帧保存的一次图文反馈。

每条卡片建议展示：

1. 标注图片缩略图
2. 点击查看大图
3. 说明文字
4. 帧号 / 时间码
5. 对应版本号
6. 来源审片任务
7. 提交时间
8. 导演姓名

### 7.4 关键交互

1. 点击反馈卡片查看大图
2. 点击回跳到对应版本视频的对应帧
3. 按版本筛选反馈
4. 按任务筛选反馈
5. 按时间倒序查看反馈

返修记录查看补充要求：

1. 制作人员打开镜头详情 `记录` 分页时，展开返修记录后可直接看到返修图片预览
2. 点击返修图片时可放大查看
3. 不要求制作人员依赖制片上传图片时所在机器的本地路径看图

## 8. 制作人员如何理解反馈

制作人员看到的导演反馈不应只是长文本，而应是“画面定位明确”的图文卡片。

这套形式的价值在于：

1. 能快速看出导演指出的是画面哪个位置
2. 能知道问题对应哪一帧
3. 能知道问题对应哪个版本
4. 能减少制作人员误解反馈的概率

## 9. 版本更新后的预期链路

制作人员处理反馈后的主链路建议固化如下：

1. 打开镜头详情
2. 查看导演反馈卡片
3. 结合本机可播放素材定位问题
4. 完成镜头修改
5. 产出新版本文件
6. 完成当前版本文件同步与绑定回写
7. 镜头在本轮仍保持 `待处理反馈`
8. 制作人员将镜头标记为 `已按反馈修改`
9. 等待制片重新整理并发起下一轮审片任务

说明：

1. 新版本产生后，不以“自动清空镜头二级状态”作为默认策略；需待制作人员确认本轮反馈已处理完成后再推进到 `已按反馈修改`
2. 审片任务的创建职责仍在制片，不应转移给制作人员
3. 制作人员只能处理 `已按反馈修改` 这一类二级状态动作，不负责提审组织

## 10. 与本机文件能力的关系

制作人员仍需要保留本机文件检查与预览能力。

因为：

1. 导演反馈只保存协同记录，不替代本机文件查看
2. 制作人员必须基于当前机器的文件状态确认是否已产出新版
3. 镜头详情中的可播放状态仍以客户端本地能力为准

## 11. 权限边界

制作人员账号建议具备：

1. 查看参与项目
2. 查看本人相关镜头
3. 查看镜头详情中的导演反馈
4. 查看反馈图片与说明
5. 查看镜头版本与本地素材状态
6. 将镜头标记为 `已按反馈修改`

制作人员账号不负责：

1. 创建审片任务
2. 调整任务镜头顺序
3. 修改导演反馈原始标注
4. 删除导演反馈卡片
5. 将镜头直接改为 `内部通过`
6. 将镜头直接送入 `审片中`

## 12. 制作人员端第一版范围建议

第一版建议完成：

1. 镜头列表中的导演反馈标识
2. 镜头详情中的导演反馈区
3. 二级状态展示与切换按钮
4. 反馈卡片查看与筛选
5. 回跳到对应版本与帧的能力

第二期可增强：

1. 反馈已读标记
2. 反馈处理状态标记
3. 版本前后对比查看
4. 导演反馈导出汇总

## 13. 与导演账号文档的对应关系

后续讨论制作人员功能时，必须始终核对以下原则：

1. 制作人员消费的是导演已经保存好的反馈卡片
2. 制作人员的反馈入口在镜头详情，不在导演审片工作台
3. 制作人员看到的反馈必须是图片 + 说明的结构化结果
4. 制作人员端不承担任务装配职责

## 14. 制作人员与二级状态的关系

加入导演内部审片后，制作人员端必须明确以下规则：

1. 二级状态服务于当前镜头版本
2. 当前版本升级后，不以“自动清空状态”作为默认策略；若当前处于 `待处理反馈`，则继续保持该状态直到制作人员确认处理完成
3. 制作人员完成反馈修改后，需要明确执行“已按反馈修改”动作
4. 当前二级状态未进入 `内部通过` 前，镜头不能正式提交
