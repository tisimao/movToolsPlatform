---

## 2. 本阶段目标

本阶段只做“镜头详情页导演反馈回放”和“导演审片页正式回放结果”的一致性收口，不触碰导演审片编辑链路。

完成后必须满足：

1. 同一镜头、同一版本、同一轮反馈下，镜头详情页与导演审片页看到的正式绘制内容一致
2. 版本视频优先、Layout 兜底、无视频降级提示三种来源语义一致
3. 镜头详情页导演反馈分页的交互文案、数据来源说明、异常提示完成收口
4. 本阶段结束后，可将该链路纳入稳定基线

---

## 3. 工作边界

1. 本阶段只允许调整镜头详情页回放展示链路、回放文案和只读 helper
2. 本阶段不改导演审片页的绘制编辑、草稿提交、反馈提交主流程
3. 本阶段不改服务端 DTO
4. 本阶段若需要补充测试脚本或页面级自测辅助，只能服务于回放一致性验证

---

## 4. 推荐重点文件

- `src/pages/LensPage.tsx`
- `src/components/lens/DirectorFeedbackPlayback.tsx`
- `src/lib/reviewPlayback.ts`
- `src/lib/reviewFrameFeedback.ts`
- `src/styles.css`

---

## 5. 必做事项

1. 对照导演审片页正式回放路径，核对镜头详情页在“版本过滤、轮次过滤、帧号解析、视频源解析、绘制解算”五个环节是否完全一致
2. 统一镜头详情页中的播放源文案，让用户能区分“版本视频”“Layout 兜底”“当前无可回放视频”
3. 统一镜头详情页导演反馈分页的异常提示，避免再出现“有反馈卡片但无绘制来源”却无解释的情况
4. 增加必要的只读日志或调试说明时，必须保持默认用户界面简洁，不得把调试文本长期暴露在主界面
5. 回归验证至少覆盖：单版本单轮、多版本、多轮、仅 Layout、无视频五类场景

---

## 6. 禁止事项

1. 禁止改导演审片工作台的绘制、暂停标注、草稿、提交主链路
2. 禁止扩散到镜头列表、任务列表等无关模块
3. 禁止把“镜头详情页显示一致”建立在复制一份导演审片复杂编辑逻辑之上
4. 禁止用隐藏开关或临时选项规避正式一致性问题
5. 禁止跳过多版本/多轮场景验证

---

## 7. 验收标准

1. 同一镜头在导演审片页与镜头详情页看到的正式绘制结果一致
2. 多版本情况下，镜头详情页不会串用其他版本反馈回放数据
3. 多轮情况下，镜头详情页只显示当前版本当前轮正式绘制
4. 版本视频优先、Layout 兜底、无视频降级提示清晰可读
5. `npm run typecheck` 与 `npm run build` 通过
