# 平台功能与UI统一命名清单

## 1. 文档目的

本文件用于统一当前平台的功能称呼、页面名称、左侧菜单名称、角色可见菜单范围与常用 UI 区块命名，作为后续提需求、派任务、做验收时的唯一用词基线。

后续描述界面调整时，优先使用“页面名 + 区块名 + 角色”表达，例如：

- 制片账号的“项目”页 - “项目成员管理（制片）”区块
- 制作人员账号的“仪表盘”页 - “我可访问的项目”区块
- “左侧主导航”中的“镜头”菜单

## 2. 顶层命名基线

### 2.1 产品与客户端称呼

1. 产品可统称为“Movtools 协同平台”
2. 当前桌面端客户端正式称呼为“Movtools 协同客户端”
3. 登录后的左侧菜单统一称为“左侧主导航”
4. 左侧主导航中的每一项统一称为“一级导航菜单”
5. 页面头部的大标题区域统一称为“页面头部”
6. 界面中 `Layout` 业务名词统一保留为 `Layout`，不再改写为其他中文别名

### 2.2 页面正式名称

当前已实现页面正式名称如下：

1. 登录页
2. 仪表盘
3. 项目
4. 文件检查
5. 提取
6. 镜头
7. 审片
8. 工作台
9. 日志
10. 设置
11. 用户
12. 路径映射设置（嵌入在“设置”页高级功能中，不是一级导航）

## 3. 左侧主导航统一命名

左侧主导航的一级菜单名称统一如下：

1. 仪表盘
2. 项目
3. 文件检查
4. 提取
5. 镜头
6. 审片
7. 工作台
8. 日志
9. 设置
10. 用户

补充说明：

1. “用户”菜单仅系统管理员可见
2. “路径映射设置”不出现在左侧主导航中，正式入口在“设置”页
3. 虽然系统内部仍使用英文稳定码，但所有需求描述、验收口径、界面沟通统一使用上述中文菜单名
4. `Layout` 在界面展示中统一使用 `Layout`，例如“Layout 协作”“Layout 视频”“Layout 根目录”

## 4. 角色与菜单可见范围

### 4.1 制片账号登录时可见菜单

制片账号登录后，左侧主导航可见：

1. 仪表盘
2. 项目
3. 文件检查
4. 提取
5. 镜头
6. 审片
7. 工作台
8. 日志
9. 设置

补充说明：

1. 制片账号当前没有独立“用户”一级菜单入口
2. 制片的项目成员管理主入口在“项目”页内，不在“用户”页内

### 4.2 制作人员账号登录时可见菜单

制作人员账号登录后，左侧主导航实际收口为：

1. 仪表盘
2. 镜头
3. 设置

补充说明：

1. 制作人员虽然底层有部分页面访问能力定义，但当前客户端界面已将左侧主导航收口为以上 3 项
2. 制作人员的工作入口以“仪表盘”和“镜头”为主
3. 制作人员需要的路径映射入口位于“设置”页里的高级功能

### 4.3 其他角色菜单基线

1. 系统管理员：仪表盘、项目、文件检查、提取、镜头、审片、工作台、日志、设置、用户
2. 导演：仪表盘、镜头、审片
3. 只读查看者：仪表盘、工作台、日志、设置

补充说明：

1. 导演账号的左侧主导航已正式收口为“仪表盘 / 镜头 / 审片”
2. “项目 / 工作台 / 日志 / 设置”不再作为导演主导航入口
3. 导演查看项目上下文、任务入口与最近反馈，应优先在“导演仪表盘”中完成

## 6. 后续需求描述推荐句式

后续提需求时，建议直接使用以下句式：

1. 调整“左侧主导航”中“镜头”菜单的排序或文案
2. 调整制片账号在“项目”页“项目成员管理（制片）”区块的交互
3. 调整制作人员账号在“仪表盘”页“我可访问的项目”卡片的展示字段
4. 调整“文件检查”页“当前集 Layout 协作区”的导出按钮文案
5. 调整“工作台”页“镜头池”或“拼接顺序”区块布局
6. 调整导演账号在“仪表盘”页“当前激活项目的审片任务”区块的字段展示
7. 调整制片账号在“审片”页“审片任务管理”区块的任务装配交互
8. 调整导演账号在“审片”页“任务镜头队列”区块的切换交互

## 7. 使用约定

1. 后续新增页面、菜单、弹窗、主区块时，必须先补充本文件
2. 若页面显示名与导航名不一致，以本文件更新后的正式名称为准
3. 若角色权限变更导致菜单可见范围变化，必须同步更新本文件与 `通信协议文档.md`
