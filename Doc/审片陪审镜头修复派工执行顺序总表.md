## 2. 本轮问题背景

本轮修复聚焦以下同类问题：

1. `context` 陪审镜头在导演工作台中被错误当成 `review` 正式镜头
2. 陪审镜头仍显示 `镜头通过` 和 `当前帧反馈` 正式编辑区
3. mixed task 中陪审镜头错误参与完成判定，导致 `审片完成` 不可点击
4. 制片账号点击错误入口时触发导演接口 403
5. 之前派工包已改一部分 UI 与状态机，但旧入口仍可能把 `ParticipationMode` 默认写成 `review`

## 4. 推荐执行顺序

1. 先执行后端派工包
2. 后端完成后执行前端派工包
3. 前后端都完成后执行数据修复与回归测试派工包
4. 每阶段结束后都要回填修改文件、规则变化、自测结果与剩余风险

## 5. 派发要求

1. 一次只派发一份文档，不要把三份指令包混成一条长指令同时执行
2. 执行方必须先阅读 `项目固化文档.md` 和 `通信协议文档.md`
3. 若第一阶段未修掉旧入口默认 `review` 的问题，不得直接跳到第二阶段
4. 若第三阶段发现历史脏数据存在，必须把“代码已修复”和“数据已修复”分开汇报
5. 每阶段都必须给出不能复发的依据，而不是只给功能通过截图
